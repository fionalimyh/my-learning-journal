# Reflection Journal
### Write a short note (200 words or less) answering these two questions:

"What is one thing about Git or GitHub that I now understand and could explain to a friend?"
I understand that Git helps me track changes made to my files, while GitHub stores those files online so I can back them up and access them easily. I also now know the purpose of the workflow: first I make changes in VS Code, then stage the file, write a commit message to describe what I changed, and finally push the changes to GitHub. This helps me keep a clear record of my work and makes it easier to manage updates over time. It works for communications from Github to VS Code as well. 

"What is one thing I'm still unsure about, and what is my plan to figure it out?"
One thing I am still unsure about is how branching and pull requests work in real projects. I understand they are used to manage changes safely, but I am not yet confident about when and why to use them. My plan is to first practise the core workflow until I am comfortable with it, then explore branching and pull requests by watching beginner tutorials and trying them in a small test repository on GitHub.
