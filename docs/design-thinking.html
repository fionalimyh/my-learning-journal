The Flow

Building the Portfolio

This portfolio was designed with UX-first thinking — starting from user goals, mapping a content hierarchy, wireframing layouts with Bootstrap 5.

01
Define Goals & Audience

Target audience: Tech Recruiters and Hiring Managers.
Goal: Convey technical depth, leadership and skills through Portfolios.
Hook: "building systems that scale"; "integrating GenAI into system workflows".

02
Information Architecture

Home: Gives users the first impression, crucial in determining drop-offs.
Design Thinking: Shows the methodology of building the Portfolio site.
About: Shows credibility through the education, experience and suporting details.
Portfolio: Shows the past projects and a contact form to connect.

03
Wireframe & Layout Planning

Simple wireframes sketches: Identified hero placement, card grid density, and CTA positioning before coding.

04
Bootstrap 5 Implementation

Utilised Bootstrap5's grid: col-12 / col-md-6 / col-lg-3 for the cards.
Custom CSS layering: for dark theme, effects and typography.l

05
Test & Iterate Responsiveness

Validated the RWD breakpoints: Mobile (<576px), Tablet (576–992px) and Desktop (≥992px).
Further customisations: Adjusted typography scale, stat bar layout and widths per breakpoint.

Key Design Decisions

COLOUR
Background · Text · Accent Palettes
A dark navy background with glass cards for depths. Consistent text and accent palettes for futuristic tech theme.
TYPOGRAPHY
Syne · Outfit · DM Mono
The trending tech font, Syne. Outfit font is a highly legible body. DM Mono excudes a technical feel when applied to labels and tags — reinforcing the Tech-Ops theme.
MOTION
Animations · Effects
Typewriter on hero signals activity. Scroll-reveal displays content progressively. Hover effects (card lift, glow) gives interactive feedback to users.

Jakob 10 Heuristics · Analysis

Heuristic Evaluation

Two portfolio sites evaluated against Nielsen's 10 Usability Heuristics.
Use the tabs to switch between references, and click any row to read the full findings.

Website URL: Fluid.glass
A glass glazing website showcasing designs with media integrations, interactive scrolling transitions and bold typography.

Usage of familiar designs
The progress bar at the bottom of the slider on the Projects page alerts users before the image changes.


Additionally, there is a hover function that shows "VIEW" when user hovers over active elements. This lets users know that the element is clickable if they are interested in learning more about the product.

Usage of familiar designs
Using images of "Doors", "Windows and "Structural" which are familiar to content readers and allow them to quickly identify their required service immediately. Furthermore, displaying the keyword over the image to further support the user's thought process to land on the correct inference immediately.

Unable to return to previous page
There is no dedicated 'Back' button available on the interface. Once users navigate to a new page, they must rely on the browser's back function which limits navigation freedom.

Consistency in designs and theme
Using a consistent arrow design in buttons allows users to easily identify them.


Likewise, the consistent typography and colour theme keeps the website tidy and uncluttered. This helps to reduce the user's cognitive load.

Designs prevent user mistakes
The contact form includes required field indicators and basic validation.


However, the error message only displays upon clicking the 'Submit' button, which increases user friction.

Floating menu reduces memory load
The floating menu at the bottom of the page remains visible as the user navigates the site. It minimises the user's memory load by making navigation options constantly visible.

Filter function available for expert users
On the Projects page, a filter function is available to help expert users quickly select a specific product rather than scrolling through all the projects. This acts as an accelerator to expert users.

Alignment issues
There is a consistent use of colour theme and typography but the alignment of images are off on the home page.


In addition, there are inconsistent gaps or missing cards on the project page.

Error messages are clear and concise
The error messages displayed for missing or incorrect fields are clear and specify the exact problem. It provides constructive feedback which allows the user to easily correct the error.


However, the system fails to display an error message when a user uploads an invalid file.

FAQ is available
The FAQ is available if user requires any help.
Overall Rating Pass: 6/10 · Neutral: 2/10 · Needs Improvement: 2/10

Website URL: Deduxer.studio
A motion-design and creative studio portfolio — dark-mode first, heavily interactive with scroll-triggered animations and rich user interactions.

Real-time feedback on scroll and interaction
Scroll progress indicators on the right integrated into the animated transitions helps to communicate page position effectively.


There is hover function that shows "Click" when user hover above dynamic elements and "Scroll" for static elements. These allow users to know that there is additional hidden information under the click.


Universal Icons
The use of universal icons help users to identify the content information immediately. This helps reduce the cognitive and mental load of users.

Users may feel locked into scroll experiences
Although the transition effects are visually appealing, there is no skip-to-content button. There are irrelevant animations that do not value-add to the content.


Additionally, there is no dedicated 'Back' button available on the interface. Once users navigate to a new page, they must rely on the browser's back function which limits navigation freedom.

Overloaded transition effects
Using consistent black buttons allow users to easily identify them and help reduce their cognitive load.


Despite having consistent colour schemes and typography, the oversized headers and overloading transitions can cause confusions to users.

Designs prevent user mistakes
The contact form includes required field indicators and basic validation. The error message for email displays immediately when the user did not fill in the correct format.

Floating menu reduces memory load
The floating header at the top of the page remains visible as the user navigates the site. It minimizes the user's memory load by making navigation options constantly visible.

Shortcut available for expert users
On the Cases page, there is a menu function available to help expert users quickly select the specific case rather than scrolling through all the cases. This acts as an accelerator to expert users.

Poor positioning and padding
On Pricing page, the contents have minimum to no padding. The page looks extremely cluttered with lines from table and overlapping cards of different opacity. There are mixtures of font sizes and misalignment of elements which can cause cognitive overload.

Error messages are clear and concise
The error messages displayed for missing or incorrect fields are clear and specify the exact problem. It provides constructive feedback which allows the user to easily correct the error.

FAQ is available
The FAQ is available if user requires any help.

Responsive Web Design

RWD Implementation

This site meets all three required breakpoints using Bootstrap 5's grid system combined with custom CSS media queries.

📱
Mobile
< 576px
  • Single-column layout (col-12)
  • Full-width CTA buttons
  • Stats bar stacks vertically
  • Hamburger menu visible
  • Orbs hidden to save performance
  • Avatar scaled to 210×270px
💻
Tablet
≥ 576px · < 992px
  • Two-column card grid (col-md-6)
  • Hero text at 2.7rem scale
  • Avatar 250×320px
  • Hamburger menu still visible
  • Stats bar adjusts min-width
  • Section padding reduced to 64px
🖥
Desktop
≥ 992px
  • Four-column card grid (col-lg-3)
  • Hero side-by-side layout
  • Full nav links (no hamburger)
  • All decorative orbs visible
  • Avatar full 300×380px
  • Hero padding 80px

Bootstrap 5 Grid Classes Used

The site uses Bootstrap's responsive tier system throughout. Below is a summary of the main breakpoint classes applied per component:

SKILLS / ABOUT GRID
